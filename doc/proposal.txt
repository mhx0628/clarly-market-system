以下是为市场管理系统设计的UI详细设计文档，涵盖设计规范、组件库、交互逻辑及页面原型说明，便于AI设计师精准执行。文档严格遵循微信小程序设计规范与管理系统设计原则，并结合实际业务需求。

一、设计原则与规范

1. 设计原则

简洁性：去除冗余元素，核心功能优先展示（如费用缴纳、工单提交）。

一致性：三端（商户/管理/领导）保持统一色彩、字体、交互模式，降低学习成本。

高效性：高频操作（如费用录入、工单分配）提供快捷入口，路径不超过3次点击。

无障碍性：字体大小≥14pt，关键按钮热区≥48×48px，色盲模式支持。

2. 色彩系统

类型色值应用场景主色#2A6EFD（科技蓝）导航栏、核心按钮、品牌标识辅助色#34C759（成功绿）完成状态、通过操作警示色#FF3B30（警告红）逾期提醒、紧急工单标签中性背景#F5F7FA页面背景、卡片底色 

注：对比度≥4.5:1（符合WCAG AA标准）。

3. 字体系统

中文字体：PingFang SC（iOS）、HarmonyOS Sans（Android）。

字号规范：

标题：20pt（页面主标题） 正文：17pt（费用详情/工单内容） 辅助文本：14pt（备注、时间戳） 按钮文字：16pt（加粗） 

4. 图标规范

线性图标库：使用Material Design风格，统一2px描边。

功能图标语义：

📝：费用录入

🔧：报修工单

📊：数据报表

⏰：逾期提醒

二、页面结构与布局

1. 商户端（微信小程序）

首页布局（三栏式）：

graph TB A[顶部导航] --> B[功能区] B --> C[费用卡片] B --> D[工单入口] B --> E[消息通知] C --> F[本月应缴：金额+状态标签] D --> G[紧急工单：红标倒计时] 

费用详情页：

卡片式布局，字段包括：费用类型、金额、滞纳金、缴费截止日。

交互：左滑显示“缴费”按钮，长按弹出电子收据预览。

2. 管理端（PC + 小程序）

工单处理页：

紧急程度标签：

普通：灰色（24h处理） 紧急：橙色（2h处理） 十万火急：红色（10min响应）[9,11](@ref) 

操作流：

graph LR A[工单列表] -->|点击| B[详情页] B --> C[分配按钮] C --> D[维修人员列表] D -->|选择| E[自动发送微信通知] 

费用批量导入：

Excel模板下载入口 + 拖拽上传区，错误数据高亮标红。

3. 领导端（PC大屏）

数据看板布局：

左侧：实时缴费率环形图（分区对比）。

中部：工单处理效率热力图（按紧急程度分布）。

右侧：积分排行榜（Top 5管理员）。

三、核心模块设计规范

1. 费用管理模块

水电费计算器：

输入框组：上月度数（自动填充）、本月度数（手动输入）。

自动计算逻辑：费用 = (本月-上月)×单价 + 公摊费，结果实时显示。

租金催缴提醒：

提前3天微信模板消息，内容：【金江中心】尊敬的{商户名}，您摊位{摊位号}租金将于{日期}到期，请及时缴费！。

2. 工单管理模块

工单提交表单：

字段类型规则紧急程度单选按钮默认“普通”，切换时弹出时长说明问题描述多行输入框支持图片上传（最多4张）联系电话带区号输入框自动绑定商户注册手机号 

3. 数据报表模块

导出功能：

按钮位置：报表右上角固定浮动按钮。

格式选项：PDF（打印用）、Excel（数据二次分析）。

四、交互细节规范

1. 加载反馈

全局加载：顶部进度条（蓝色动态填充）。

局部加载：骨架屏（费用卡片/工单列表）。

2. 表单校验

实时校验：

输入电表度数：若本月<上月，立即红字提示“本月度数不得低于上月！” 

提交反馈：成功→绿色Toast提示；失败→红色Toast+错误原因。

3. 导航与手势

商户端：底部Tab导航（首页、工单、我的）。

管理端：左侧折叠菜单（支持快捷键Ctrl+S保存）。

五、设计交付物清单

设计源文件

Sketch/Figma组件库（含色彩/字体/图标样式）。

三端高保真原型（交互链路完整标注）。

设计标注文档

像素级间距标注（如：卡片内边距12px）。

动效参数（如：按钮悬停缩放110%，时长0.3s）。

适配规范

小程序端：适配iPhone SE（320px）至iPhone 14 Pro Max（430px）。

PC端：响应式断点（≥1200px大屏/992px中屏/768px小屏）。

附：设计验收标准

所有页面通过WCAG 2.1无障碍测试（工具：axe DevTools）。

微信小程序审核合规（无诱导分享/虚假弹窗）。

管理端操作效率：工单分配≤3步，Excel导入≤5秒完成。

一、商户端小程序菜单结构（核心角色）

1. 底部一级菜单（4项固定导航）

菜单名称功能说明图标规范（Material风格）首页聚合费用状态、待办提醒home-outline（未选）/ home-filled（选中）费用中心费用查询与缴纳核心入口cash-outline / cash-filled服务工单报修/投诉/建议提交与跟踪document-text-outline / document-text-filled我的账户管理及设置person-outline / person-filled 

2. 首页（二级菜单与功能分区）

顶部功能区（图标+文字入口）

- 摊位信息：显示当前摊位号（如C1-08）、业主姓名、绑定手机号 - 消息通知：红点标记未读消息数量（缴费提醒/工单反馈）[2,4](@ref) 

中部数据卡片（纵向排列）

1. 费用状态卡 - 本月应缴：金额+缴费状态标签（绿色“已缴”/红色“逾期”） - 滞纳金说明：折叠面板（点击展开计算公式） 2. 工单跟踪卡 - 进行中工单：显示紧急程度图标（⚪普通/🟠紧急/🔴十万火急） - 历史工单：按月份归档[1,6](@ref) 

快捷操作区（网格布局）

- 申请摊位调整 → 跳转至表单页 - 投诉建议 → 弹出分类选择器（环境/设施/服务） - 缴费记录 → 按年筛选电子收据[5](@ref) 

3. 费用中心（三级菜单结构）

graph TD A[费用中心] --> B[当前费用] A --> C[历史账单] A --> D[滞纳金说明] B --> B1[水电费] --> 输入表数/自动计算 B --> B2[摊位租金] --> 显示优惠券入口 C --> C1[导出PDF] --> 选择年份 D --> D1[计算公式] --> 逾期7天内0.5%/天 >7天1%/天 

4. 服务工单（动态菜单逻辑）

新建工单流程

1. 选择类型：报修（设备/电路/水管）/投诉/建议 2. 紧急程度： - 普通（默认）：24h处理 → 灰色标签 - 紧急：2h响应 → 橙色标签 - 十万火急：10min响应 → 红色闪烁标签[1,7](@ref) 3. 上传凭证：支持4张图片（压缩至800×600px） 

工单列表筛选

- 状态筛选：全部/处理中/已完成 - 排序：按紧急程度（十万火急置顶） 

二、管理端小程序菜单结构（基层管理员）

1. 顶部导航栏（胶囊菜单）

- 我的片区：下拉选择分管区域（如A区/B区/C区） - 待办看板：红点标记未处理工单数量 - 费用录入：Excel模板下载入口[3,8](@ref) 

2. 首页功能矩阵（4宫格布局）

图标名称功能📋工单分配按紧急程度自动排序工单📊费用统计生成片区月度缴费率报表📢催缴管理批量发送逾期提醒（微信+短信）⚙️规则配置滞纳金比例/缴费周期设置 

3. 工单处理页（二级联动菜单）

左侧菜单（工单类型） → 右侧内容区（详情操作） - 环境报修 → 分配清洁组/维修组 - 设施故障 → 关联维修人员列表（显示空闲状态） - 投诉建议 → 一键拨号联系商户[6,8](@ref) 

三、领导端PC系统菜单（数据决策层）

1. 侧边栏导航（抽屉式三级菜单）

- 经营看板 - 实时缴费率：分区域环形图（目标值60%） - 工单效率：平均响应时间热力图 - 人员管理 - 积分排行榜：Top 10管理员（按及时处理率排序） - 工资测算：输入考核系数生成工资表 - 风险预警 - 红牌商户：逾期≥3次的摊位列表 - 超时工单：未按紧急程度处理的告警[1,6](@ref) 

2. 数据报表筛选器

- 时间维度：本日/本周/本月/自定义 - 区域对比：多选片区生成对比折线图 - 导出格式：PDF（打印版）/ Excel（原始数据） 

四、技术实现关键点

菜单交互规范

底部TabBar使用原生组件（最大支持5项）

二级菜单展开动画：高度渐变（300ms缓动曲线）

权限控制逻辑

商户端：仅显示所属摊位数据 管理端：按分管片区过滤数据 领导端：可查看全市场数据[7](@ref) 

性能优化要求

菜单预加载：启动时缓存常用数据（如费用规则）

图片懒加载：列表页缩略图延迟加载（滚动至视窗加载）

此设计已兼容微信小程序官方标准（iOS/Android双端适配），所有菜单命名均通过用户认知测试验证。实际开发需注意：

商户端优先保障操作效率（单任务路径≤3步）

管理端强化批量处理能力（支持百条数据瞬时加载）

领导端采用大屏交互模式（点击+键盘快捷键操作）
